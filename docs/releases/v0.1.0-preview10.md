# v0.1.0-preview10 Release Notes

Tag: `v0.1.0-preview10`  
Date: 2026-02-22

## Summary

- Fixed setup/runtime bridge panic class seen during interactive setup flows.
- Refactored sync->async runtime boundaries across setup/tools/plugins.
- Improved setup sensitive-input UX across channels (hidden/visible input mode, clearer prompt behavior).

## Breaking Changes

- None expected for typical local/operator usage.

## Migration Steps

1. Backup before upgrade:
   - `cara backup --output ./carapace-backup.tar.gz`
2. Upgrade to `v0.1.0-preview10`.
3. Verify baseline outcomes:
   - `cara verify --outcome auto --port 18789`
   - `cara verify --outcome autonomy --port 18789`

## Rollback Steps

1. Stop Cara.
2. Reinstall previous known-good preview tag.
3. Restore backup:
   - `cara restore --path ./carapace-backup.tar.gz`
4. Re-run verification:
   - `cara status --port 18789`
   - `cara verify --outcome auto --port 18789`

## Security

- Removed panic-prone runtime-bridge patterns that could break setup flows under nested runtime contexts.
- Hardened runtime-bridge call paths and regression coverage for sync->async boundary handling.
- Improved sensitive input handling/clarity in setup UX to reduce accidental secret exposure risk.
