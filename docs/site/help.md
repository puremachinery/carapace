# Get Help

## Outcome

Get unstuck on setup, request a small-team evaluation, or point us at the next high-value workflow to document.

## Before you open a request

Use the shortest matching doc path first:

- [Install](install.md)
- [First Run](first-run.md)
- [Providers Hub](providers.md)
- [Get Unstuck](get-unstuck.md)
- [Cookbook](../cookbook/README.md)

If those still leave you blocked, use one of the request paths below.

<a id="guided-setup"></a>
## Guided setup help

Use this when you want help getting to a concrete first outcome such as local chat, Discord, Telegram, hooks, or the OpenAI-compatible endpoint.

- Best for: individual operators and self-hosted first runs
- Include: version, OS, provider path, desired outcome, and the first failing step
- Keep it public-safe: no secrets or private customer data

[Open a guided setup help request](https://github.com/puremachinery/carapace/issues/new?template=setup-help-request.yml&title=setup+help%3A+%3Cgoal%3E)

<a id="team-pilot"></a>
## Team setup / pilot request

Use this when you are evaluating Carapace for a small team, want guided setup or hardening help, or want to talk through a managed pilot at a high level.

- Best for: team evaluations and repeatable setup-assist conversations
- Include: use case, team size, deployment expectations, blockers, and timeframe
- Keep it public-safe: high-level only; we can move deeper follow-up off-thread if needed

[Open a team setup / pilot request](https://github.com/puremachinery/carapace/issues/new?template=team-pilot-request.yml&title=team+pilot%3A+%3Cteam+or+use+case%3E)
